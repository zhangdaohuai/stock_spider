# Smart Search 改进计划（Playwright 方案）

## 问题分析

当前测试失败的主要原因：

1. **网络限制**：多个财经网站（如新浪财经、雪球、同花顺）返回 301/404/502 错误
2. **反爬措施**：Google 搜索触发 reCAPTCHA 验证码
3. **数据获取**：无法从公开网站获取到有效的财务报告信息
4. **存储功能**：缺少数据存储到数据库和文件系统的完整实现

## 解决方案

### 1. 数据源优化

**主要数据源**：
- **财经门户网站**：东方财富网、新浪财经、雪球、同花顺
- **公司官方渠道**：海天味业官网投资者关系栏目
- **交易所公告**：深交所/上交所公告系统

**技术实现**：
- 使用 Playwright 浏览器自动化
- 实现反爬策略
- 多网站并行抓取

### 2. 技术架构调整

**核心组件**：
1. **数据采集层**：
   - Playwright 浏览器自动化
   - 多网站爬虫
   - 反爬策略优化

2. **数据处理层**：
   - HTML 解析与数据提取
   - 财务数据结构化
   - 文档下载与解析

3. **存储层**：
   - 数据库存储（PostgreSQL）
   - 文件存储（本地文件系统）
   - 缓存系统（Redis）

4. **API 层**：
   - 统一数据访问接口
   - 数据查询与检索

### 3. 实施步骤

#### 步骤 1：增强 Playwright 浏览器自动化
- 优化浏览器配置，避免被识别为爬虫
- 实现智能等待和页面交互
- 配置合适的请求头和用户代理

#### 步骤 2：实现多网站爬虫
- 东方财富网爬虫：获取公司财报和新闻
- 新浪财经爬虫：获取财经新闻和数据
- 雪球爬虫：获取用户讨论和分析
- 同花顺爬虫：获取财务数据和指标

#### 步骤 3：实现文档下载功能
- 识别并下载 PDF/Excel 文档
- 实现文档存储到指定位置
- 支持文档内容提取

#### 步骤 4：实现数据存储
- 设计数据库 schema 存储财务数据
- 实现文件存储系统
- 建立数据索引和检索机制

#### 步骤 5：优化搜索策略
- 实现多网站并行搜索
- 智能结果排序和去重
- 增强错误处理和容错机制

## 具体实施计划

### 技术实现

1. **爬虫模块**：
   - `smart_search/core/crawlers/playwright_crawler.py`：基础爬虫
   - `smart_search/core/crawlers/eastmoney.py`：东方财富网爬虫
   - `smart_search/core/crawlers/sina.py`：新浪财经爬虫
   - `smart_search/core/crawlers/xueqiu.py`：雪球爬虫
   - `smart_search/core/crawlers/10jqka.py`：同花顺爬虫

2. **存储模块**：
   - `smart_search/core/storage/db_storage.py`：数据库存储
   - `smart_search/core/storage/file_storage.py`：文件存储

3. **工具模块**：
   - `smart_search/core/utils/document_parser.py`：文档解析
   - `smart_search/core/utils/data_extractor.py`：数据提取
   - `smart_search/core/utils/anti_crawl.py`：反爬策略

4. **配置模块**：
   - `smart_search/config/crawler.py`：爬虫配置
   - `smart_search/config/storage.py`：存储配置

## 风险与应对

1. **反爬措施**：
   - 应对：实现随机用户代理、请求间隔、浏览器指纹伪装

2. **数据质量**：
   - 应对：多源数据交叉验证，建立数据质量评估机制

3. **存储容量**：
   - 应对：实现数据压缩和清理策略

4. **性能问题**：
   - 应对：优化并发处理，实现缓存机制

## 技术要点

1. **Playwright 优化**：
   - 使用无头浏览器模式
   - 实现智能等待和页面交互
   - 配置合适的浏览器参数

2. **反爬策略**：
   - 随机用户代理
   - 请求间隔控制
   - 浏览器指纹伪装
   - 验证码处理（如需要）

3. **数据提取**：
   - 使用 CSS 选择器和 XPath
   - 实现结构化数据提取
   - 处理动态加载内容

4. **文档处理**：
   - 识别文档链接
   - 下载并存储文档
   - 解析文档内容
