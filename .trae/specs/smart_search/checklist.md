# smart_search 模块 - 验证清单

## 功能验证
- [ ] 检查多引擎适配器是否正常工作，能够切换使用 Tavily AI 和 SearXNG
- [ ] 检查意图识别与 Query 自动补全功能，特别是 finance 主题的查询优化
- [ ] 检查深度内容解析模块，能够抓取网页内容并转换为 Markdown
- [ ] 检查来源可靠性评分模块，能够对不同来源给出正确的评分
- [ ] 检查结构化输出模块，返回的 JSON 数组符合指定格式
- [ ] 检查文件下载模块，能够下载上市公司公告到本地指定目录
- [ ] 检查数据库和缓存集成，能够连接 PostgreSQL 和 Redis
- [ ] 检查数据源管理，能够确定数据源并存储到单独的表中
- [ ] 检查异常分级和邮件通知，关键性错误能够通过邮件告知
- [ ] 检查数据库设计和更新技能集成，能够自动进行数据库版本更新

## 性能验证
- [ ] 检查基础搜索响应时间是否在 1 分钟内
- [ ] 检查深度搜索响应时间是否在 3 分钟内
- [ ] 检查网页抓取阶段是否使用 asyncio 并发请求
- [ ] 检查是否使用代理池和随机 User-Agent

## 异常处理验证
- [ ] 检查异常处理逻辑，能够处理网络错误、超时等异常
- [ ] 检查空结果兜底机制，当搜索无结果时返回建议的搜索词
- [ ] 检查日志记录是否完整且结构化

## 测试用例验证
- [ ] 测试用例 1: 搜索"2024年半导体行业增长率"，depth='basic'，应返回不少于 5 条相关新闻链接及摘要
- [ ] 测试用例 2: 搜索"茅台 2023 财报"，file_type=['pdf']，应优先返回巨潮资讯或官网的 PDF 链接
- [ ] 测试用例 3: 验证时间过滤，设置 days_limit=7，结果集中所有 publish_date 必须在 7 天内
- [ ] 测试用例 4: 验证 Markdown 转换，full_content 中不得包含 <script> 或 <div> 标签
- [ ] 测试用例 5: 验证文件下载功能，能够下载不同格式的上市公司公告

## 代码质量验证
- [ ] 检查代码结构是否清晰，符合 Python 最佳实践
- [ ] 检查注释是否完整，文档是否齐全
- [ ] 检查依赖管理是否正确，无冗余依赖
- [ ] 检查配置管理是否合理，支持不同环境的配置

## 部署验证
- [ ] 检查环境变量配置是否正确
- [ ] 检查数据库和缓存连接是否正常
- [ ] 检查文件存储目录是否正确创建和权限设置
- [ ] 检查服务启动和运行状态是否正常
