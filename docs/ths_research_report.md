## 一、研究概述

本报告针对同花顺(10jqka.com.cn)的数据获取能力进行全面研究，重点验证**历史分钟线数据**的获取方案，同时分析所有可获取的数据内容和反爬策略。

> **⚠️ 项目范围限定**：本项目**仅关注沪深主板A股**，完全放弃创业板(30xxxx)、科创板(68xxxx)、北交所(83/87xxxx)数据。所有规则和方案均以主板为基准。

### 核心结论

| 方案 | 1分钟线 | 5分钟线 | 历史深度 | 免费 | 推荐度 |
|------|---------|---------|---------|------|--------|
| **同花顺JSONP v6 API** | ✅ | ❌ | 2026年3月至今 | ✅ | ⭐⭐⭐⭐ |
| **同花顺JSONP v2 API** | ❌ | ⚠️不稳定 | 30/60分钟线2024+ | ✅ | ⭐⭐⭐ |
| **THSDK (C库封装)** | ⚠️需账号 | ⚠️需账号 | 取决于账号权限 | 游客受限 | ⭐⭐⭐ |
| **mootdx (通达信协议)** | ✅(5天) | ✅(1月) | 1分钟极浅 | ✅ | ⭐⭐ |
| **pywencai (问财)** | ❌ | ❌ | 不支持分钟线 | ✅ | ⭐⭐ |

## 二、同花顺JSONP API 详细分析

### 2.1 API域名与层级

| 域名 | 用途 | 反爬等级 |
|------|------|---------|
| `d.10jqka.com.cn` | 数据接口(v2/v6) | 🟢低 |

### 2.2 v2 API Period Code 完整映射（已验证）

> **关键发现**：同花顺v2 API的period code与社区流传的映射**完全不同**！

| Period Code | 数据类型 | 每日条数 | 历史范围 | 复权变体 |
|-------------|---------|---------|---------|---------|
| 00 / 01 / 02 | 日K线 | 1 | 全量(2001+) | 前复权/后复权/不复权 |
| 10 / 11 / 12 | 周K线 | 1/周 | 全量 | 同上 |
| 20 / 21 / 22 | 月K线 | 1/月 | 全量 | 同上 |
| **30** | **5分钟线** | 48 | **不稳定** | - |
| **40 / 41 / 42** | **30分钟线** | 8 | **2024+** | 前复权/后复权/不复权 |
| **50 / 51 / 52** | **60分钟线** | 4 | **2024+** | 前复权/后复权/不复权 |
| 70 / 71 / 72 | 日K线(datetime) | 1 | 全量 | 同上 |
| 80 / 81 / 82 | 年K线 | 1/年 | 全量 | 同上 |
| 90 / 91 / 92 | 季K线 | 1/季 | 全量 | 同上 |

**URL格式**：
```
http://d.10jqka.com.cn/v2/line/hs_{code}/{period}/{year}.js
```

**数据格式**（CSV分号分隔）：
```
日期时间,开盘价,最高价,最低价,收盘价,成交量,成交额,换手率,,,0;...
```

**验证结果**：
```

```

## 四、反爬策略详细分析

### 4.1 反爬层级

| 层级 | 域名/接口 | 反爬手段 | 绕过难度 |
|------|----------|---------|---------|
| 🟢低 | d.10jqka.com.cn (v2/v6) | Referer + User-Agent | 极易 |
| 🟡中 | q.10jqka.com.cn (第2页+) | chameleon.js动态Token | 中等 |
| 🔴高 | www.iwencai.com | hexin-v JS加密签名 + 验证码 | 困难 |
| 🔴高 | qd.10jqka.com.cn | 认证Token | 困难 |
| ⚫极高 | my.10jqka.com.cn | 登录态 + HMAC-SHA256动态sign | 极难 |

### 4.2 d.10jqka.com.cn 低反爬验证

```python
# 仅需设置Referer和User-Agent即可访问
headers = {
    "User-Agent": "Mozilla/5.0 ...",
    "Referer": "https://stockpage.10jqka.com.cn/",
}
# 无Referer: 返回404
# 无User-Agent: 返回404
# 正常设置: 200 OK ✅
```

### 4.3 q.10jqka.com.cn 中反爬验证

```
第1页: 200 OK ✅
第2页: 401 Unauthorized ❌
  → 返回chameleon.1.7.min.js验证脚本
  → 需要执行JS生成动态Token
```

### 4.4 iwencai.com 高反爬验证

```
无签名请求: 401 Unauthorized
  → 返回验证码URL: captcha_url
  → 需要hexin-v签名（JS加密生成）
  → pywencai通过Node.js执行JS绕过
```

### 4.5 高频请求测试

```
5次连续请求(间隔0.2秒): 全部200 OK
→ d.10jqka.com.cn 对高频请求容忍度较高
→ 建议仍保持1秒以上间隔以避免IP封禁
```

## 九、风险与注意事项

### 9.1 法律风险

- 同花顺`robots.txt`禁止爬取，商业使用需授权
- JSONP接口属于非公开API，随时可能变更或关闭
- 建议仅用于个人研究，商业场景购买iFinD接口

### 9.2 技术风险

- v2 API period=30(5分钟线)不稳定，有时返回404
- v6 API 1分钟线仅当年可用，历史年份不可获取
- THSDK游客模式限制严重，临时账号可能随时失效
- mootdx 1分钟线历史深度仅5天，不适合历史回溯

### 9.3 数据质量风险

- 同花顺v2/v6 API的复权方式：v2 period=01为后复权，period=00为前复权
- 分钟线数据的成交量单位：股（非手）
- 实时行情items字段映射已修正：items[69]=涨停价, items[70]=跌停价（2026-05-18验证）
- items[8]=最高价, items[9]=最低价（非涨跌停价，之前映射有误）

def get_price_limit_pct(stock_code: str, trade_date: date) -> float:
    """获取涨跌幅限制百分比"""
    if is_st_stock(stock_code):
        if trade_date < ST_LIMIT_CHANGE_DATE:

### A.6 对爬虫开发的影响

#### A.6.1 数据采集时间窗口

```
09:15  集合竞价开始(可获取集合竞价数据)
09:25  集合竞价结束(产生开盘价)
09:30  ← 连续竞价开始，1分钟线开始产生
11:30  ← 上午盘结束
13:00  ← 下午盘开始
14:57  收盘集合竞价开始
15:00  ← 收盘，当日K线完成
15:05  盘后固定价格交易开始
15:30  盘后固定价格交易结束
```

#### A.6.2 关键注意事项

1. **1分钟线采集时机**：应在15:00后采集当日完整1分钟线数据，避免数据不完整
2. **集合竞价数据**：09:15-09:25的集合竞价数据不产生1分钟K线
3. **收盘价**：14:57-15:00收盘集合竞价产生的价格才是当日收盘价
4. **T+1影响**：当日买入的股票次日才能卖出，影响实时持仓计算
5. **涨跌幅计算**：涨跌幅 = (现价 - 昨收) / 昨收 × 100%，需用昨收而非前收盘
6. **⭐ ST兼容处理**：7月6日前ST±5%，7月6日后ST±10%，代码中需动态判断（见A.2.2节）
7. **⭐ 交易日历**：调休上班日不开市，必须维护独立交易日历对象（见A.4.3节）
8. **仅主板**：本项目仅关注60xxxx/00xxxx/002xxx主板股票，排除创业板/科创板/北交所
