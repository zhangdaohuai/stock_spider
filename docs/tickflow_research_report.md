## 五、分钟线获取方案分析（基于文档）

### 5.1 方案一：直接查询历史1分钟K线

```python
# 需付费订阅
GET /v1/klines?symbol=600519.SH&period=1m&count=10000
# 单次最多10000条 ≈ 41.7个交易日
# 通过start_time/end_time分页可获取完整历史
```

**优势**：
- 完整OHLCV字段
- 支持前复权/后复权
- 单次10000条，效率高
- 可通过时间范围分页

**劣势**：
- 需付费订阅
- 当前API Key无权限

### 5.2 方案二：当日分钟K线

```python
# 需付费订阅(比完整服务更高级别)
GET /v1/klines/intraday?symbol=600519.SH&period=1m
# 仅返回当日分钟K线
```

**优势**：实时盘中数据
**劣势**：仅当日，需更高套餐

### 5.3 方案三：批量查询

```python
# 一次请求获取多只股票
GET /v1/klines/batch?symbols=600519.SH,000858.SZ&period=1m&count=10000
```

**优势**：减少请求次数，避免频率限制
**劣势**：需付费订阅

| 限制 | 说明 |
|------|------|

### 10.3 网络问题

| 问题 | 解决方案 |
|------|---------|
| 代理干扰 | 设置`session.trust_env = False` |

| SSL错误 | 关闭VPN/代理 |

## 十一、结论

1. **TickFlow免费服务的日K线质量优秀**：上市以来完整数据，5种复权方式，是日K线最佳免费数据源之一

2. **分钟线需付费**：当前API Key无分钟线权限，无法验证分钟线实际深度。根据文档描述，TickFlow的分钟线支持完整历史+复权+批量查询，是付费方案中的优质选择

3. **推荐策略**：优先使用免费方案(tdx-api + BaoStock + mootdx)，如需更高质量数据再考虑TickFlow付费订阅
